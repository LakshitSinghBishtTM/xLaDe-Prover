# Contributing to xLaDe

Thank you for your interest in contributing to **xLaDe**, an experimental Lean 4 ecosystem!
We welcome suggestions, feedback, and code contributions from the community.

## How to Contribute

1. **Fork the repository** to your GitHub account.
2. **Clone your fork** locally:
```bash
git clone https://github.com/your-username/xLaDe.git
````

3. **Create a new branch** for your contribution:

```bash
git checkout -b feature/my-new-feature
```

4. **Make your changes** in `leancore` or other relevant folders.
5. **Commit your changes** with a clear, descriptive message:

```bash
git commit -m "Add brief description of change"
```

6. **Push your branch** to your fork:

```bash
git push origin feature/my-new-feature
```

7. **Open a Pull Request** on the main repository. Include:

* Summary of your changes
* Motivation / why itâ€™s useful
* Any known limitations or caveats

## Reporting Issues

* Use the **GitHub Issues** tab to report bugs, request features, or suggest improvements.
* Provide as much detail as possible, including:

  * Steps to reproduce (if applicable)
  * Expected vs. actual behavior
  * Relevant logs or code snippets

## Code of Conduct

By participating in xLaDe, you agree to follow our **Code of Conduct**, which encourages:

* Respectful communication
* Constructive feedback
* Inclusivity and collaboration

## Notes

* xLaDe is an **experimental ecosystem**. Features are subject to change, and we appreciate contributions and feedback as the project evolves.
* Even small contributions, such as documentation improvements or suggestions, are highly valuable.
